Flexibox - 3 Alignment Properties

First of all, these three alignment ways are used to tell the parent container how to align its items.

justify-content

“justify-content” property is used to align items horizontally (along the x-axis) in a container.

align-items

“align-items” property is used to align items vertically (along the y-axis) in a container.

align-content

This property has no effect when there is only a single line of flex items, which contains 2 situations (maybe more):

  1. Using “flex-wrap: nowrap;” which will shink the width of items until reach the inner content width.

  2. The sum of all items’ width is still smaller than parent container’s width.

Except the two situations above, the items wrapped in the parent container (flex-wrap: wrap | wrap-reverse;) will show in multiple lines. “align-content” property is used to align these multiple lines.

But sometimes parent containers are stretched by inner items and don’t have specific height. In this case, the behavior of this property is not very obvious.

Besides, sometimes “align-content” and “align-items” show the same effect. For example: in a container with height, “align-items: center;” already places the multiple lines in space-around situation. Now “align-content: space-around;” will not change this alignment again.

Conclusion

“justify-content (x-axis)” and “align-items (y-axis)” are used to align items in one line, while “align-content” is used to align these multiple lines (if there are). Also, this is based on the default “flex-direction: row;”. If you set to “column”, the x-axis and y-axis should be reversed.
